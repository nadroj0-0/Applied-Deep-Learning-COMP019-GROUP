# Applied Deep Learning COMP0197 coursework

## Branch structure

- **`main`**: the shared branch for work thats ready and finshed.
- **Personal branches** (`your name`): your own working space, do whatever you want.

## Suggested workflow

1. Commit on your personal branch.
2. When a peice of work is ready, do a Pull Request, with main as the base branch and your personal branch as the compare branch.
3. Smaller, more frequent pull requests are probably easier to review.  
5. Avoid merging unfinished work into `main` so it doesnt get messy.

## `main` is protected

`main` is set up as a protected branch, so changes go through Pull Requests with 1 approval instead of direct pushes (one other person has a quick look before merging).  
Helps keep things tidy and so we dont accidentally break each others work.

## Quick notes

- If you ever need something from someone elses branch before its in `main`, you still have access to their branch and can download whatever you need to look through.
